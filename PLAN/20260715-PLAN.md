# 基金分红日历构建调整 Spec

## Summary

在现有“分红日历”标签页内增加“股票 / 基金”双类型切换。股票功能与 API 保持完全兼容；基金模式覆盖全部公募基金，使用东方财富全市场分红列表作为首发事件源，复用现有基金历史、公告和 AI 能力。

已确定：

- 全部公募基金，包括场外基金、ETF、LOF、REITs、FOF、QDII。
- 东方财富首发；通过 Provider 抽象预留 Tushare，不在首版引入 Token。
- 仅在币种、净值和日期口径安全时计算“本次分配比例”。
- 基金详情展示分红历史、公告证据和事件前后净值窗口。
- 首版同时提供全市场 AI 扫描和逐基金 AI 研判。
- 无数据库迁移、无新子目录、无 Nginx 调整。

## Phase 1 — 基金事件源与服务层

- [ ] 新增 `FundDividendDataProvider`、`EastmoneyFundDividendClient` 和 `FundDividendService`，沿用股票分红模块的缓存、熔断、负缓存、防击穿和 stale fallback 模式。
- [ ] 接入东方财富 `funddataIndex_Interface.aspx?dt=8`，按年份、页码、权益登记日倒序抓取；严格截取 `jjfh_data` 后使用 `json_decode`，禁止执行响应代码或使用 `eval`。
- [ ] 每页按实测上限 100 条处理，最多读取 100 页；遇到该页最早登记日早于查询开始日后停止。60 日窗口跨年时分别请求两个年度再合并。
- [ ] 以 `code + record_date + ex_date + pay_date + cash_per_unit` 去重；同一基金的 A/C/Y 类份额保留为独立事件。
- [ ] 从 `fundcode_search.js` 构建基金类型映射，归一为：
  `all|stock|index|mixed|bond|money|fof|qdii|reit|other`。
- [ ] 类型映射失败时：`fund_category=all` 可返回事件并标记 `partial`；指定类型筛选时若无 fresh/stale 映射，则返回 `metadata_unavailable`，不伪造空结果。
- [ ] 使用 `FundMNFInfo` 对需计算比例的基金进行最多 50 只/请求的批量净值查询；禁止调用现有会逐基金补详情的 `FundService::info()`。
- [ ] 仅为除息日不早于今天的候选事件补充最新净值；历史事件的分配比例留空，详情页改用除息前历史净值。
- [ ] 分配比例只在以下条件全部满足时计算：每份分红和净值均为正数、`nav_date < ex_date`、币种确认一致。
- [ ] 名称包含“美元、港币、外币、现汇、现钞”等标记时，将币种设为 `unknown`，不计算比例；其他境内份额按事件源“元/份”口径标记为 `CNY`。
- [ ] 标准化事件字段：

  - `asset_type=fund`
  - `code`、`name`、`fund_type`、`fund_category`
  - `record_date`、`ex_date`、`pay_date`、`days_to_record`
  - `cash_per_unit`、`currency`、`currency_status`
  - `nav`、`nav_date`、`nav_status`
  - `distribution_ratio_pct`
  - `ratio_status=available|missing_nav|nav_not_pre_ex|currency_unverified`
  - `event_stage=upcoming_record|upcoming_ex|payment_pending|completed`
  - `implementation_confirmed=true`
  - `announcement_match_status=not_checked`
  - `source`、`source_url`

- [ ] 新增 `fund_dividend` 配置：启用开关、默认 14 日、最大 60 日、事件缓存 900 秒、类型映射缓存 86400 秒、净值缓存 300 秒、自动刷新 900 秒及现有降级参数。
- [ ] 使用独立 `eastmoney_fund_dividend` 熔断器，避免基金事件源故障影响股票分红或其他基金接口。

## Phase 2 — 公共 API 与详情契约

- [ ] 扩展现有三个 action，新增 `asset_type=stock|fund`；省略时默认 `stock`，现有股票参数和响应不变。
- [ ] 基金日历请求：

  `market_api.php?action=dividend_calendar&asset_type=fund`

  支持：

  - `start_date`、`end_date`
  - `fund_category`
  - `min_distribution_ratio`，默认 0
  - `sort_by=record_date|distribution_ratio|cash_per_unit|pay_date`
  - `order=asc|desc`
  - `page`、`page_size`，默认 50、最大 100

- [ ] 基金默认排序为登记日升序；比例和金额为空的事件始终排在有值事件之后。
- [ ] `min_distribution_ratio > 0` 时排除无安全比例的事件；不得用缺失净值补零。
- [ ] 日历响应继续使用 `items + summary + pagination + filters`：

  - `summary.event_count`
  - `summary.unique_fund_count`
  - `summary.within_3_days_count`
  - `summary.max_distribution_ratio_pct`
  - `summary.median_distribution_ratio_pct`
  - `summary.ratio_coverage_count`

- [ ] `meta` 增加 `partial`、`failures`、`event_source`、`nav_source`、`missing_nav_count`、`ratio_coverage_count`、`upstream_cache`、`as_of`。
- [ ] 扩展 `dividend_detail&asset_type=fund&code=xxxxxx&event_date=YYYY-MM-DD`；`event_date` 可省略，省略时选择最新未完成事件，否则选择最新历史事件。
- [ ] 基金详情聚合：

  - 基金基本信息和最新净值
  - 当前选中事件
  - 完整分红历史
  - 历史事件数、覆盖年份、近五年每份累计分红
  - 最新分红公告及原文链接
  - 联接基金目标 ETF 关系说明
  - 当前事件公告匹配状态
  - 事件前后净值窗口

- [ ] 公告只有在公告正文同时匹配基金代码、登记/除息日期及分红金额时才标记 `verified`；仅成功读取公告列表时标记 `checked_unmatched`，失败为 `check_failed`，不得把“存在一篇分红公告”直接写成当前事件已核验。
- [ ] 扩展 `dividend_event_market&asset_type=fund`，复用 `code/event_date/before/after` 参数，从历史净值接口使用 `sdate/edate` 定点取窗，返回单位净值、累计净值和日增长率。
- [ ] 未来事件尚无除息后净值时返回已有前置数据，并设置 `post_event_data_pending=true`，不是错误。
- [ ] 在 `SecurityAudit` 增加基金类型、排序、资产类型和比例参数白名单；基金代码强制使用六位数字规则。

## Phase 3 — 页面与响应式交互

- [ ] 在“分红日历”标题区增加可访问的“股票 / 基金”切换控件；默认股票，并分别保存两种模式的筛选、当前页、已加载状态和请求控制器。
- [ ] 切换类型时中止旧请求并使用 request ID 防止旧响应覆盖新模式；首次切换才加载，之后恢复各自状态。
- [ ] 股票模式保持现有市场、方案状态、税档、最低毛率和排序控件。
- [ ] 基金模式改为日期窗口、基金类型、最低分配比例和排序；不显示股票市场、持有期税档、税后现金率。
- [ ] 基金汇总卡展示事件数、3 日内登记、最高安全分配比例和安全比例中位数，并显示比例覆盖条数。
- [ ] 桌面表格字段：

  `基金｜登记/除息/发放｜每份分红｜参考净值｜本次分配比例｜事件阶段｜操作`

- [ ] 移动端卡片同步展示基金类型、三类日期、每份金额、净值日期、比例状态和数据缺失说明。
- [ ] 未知币种不显示人民币符号；统一使用“每份分红”，不得写成“每股现金”。
- [ ] 状态徽章使用“待登记、待除息、待发放、已完成”；不把未来事件显示为“已实施完成”。
- [ ] 基金详情抽屉使用基金原生模板：

  - 当前事件和公告核验状态
  - 最新净值与安全分配比例
  - 完整分红历史分页
  - 公告证据列表
  - 联接基金与目标 ETF 的分层说明
  - 点选历史事件后加载单位净值/累计净值窗口图

- [ ] 基金模式提示文案明确：分红来自基金财产、净值会相应下降、不是额外或无风险收益；税务处理不沿用股票 20%/10%/0%持有期模型。
- [ ] 基金事件自动刷新仅在标签页可见且基金模式激活时执行，不受 A 股交易时段判断限制；股票刷新逻辑保持不变。
- [ ] 所有基金名称、代码、类型、公告标题、URL 属性继续使用现有 HTML/属性转义方法。

## Phase 4 — AI 工具与顾问上下文

- [ ] 新增严格 Schema 工具 `fa_get_upcoming_fund_dividends`，参数包括：

  - `start_date`
  - `days`，1–60
  - `fund_category`
  - `min_distribution_ratio`
  - `sort_by`
  - `order`
  - `limit`，1–50

- [ ] 工具调用 `FundDividendService::calendar()`，输出标准事件、覆盖率、数据时间、部分失败和来源，不自行调用其他研究工具兜底。
- [ ] AI 工具执行器增加可注入的 `FundDividendService`，作为最后一个可选构造参数，保证现有构造调用兼容。
- [ ] 更新工具注册、流式中文状态、研究状态摘要和工具覆盖标记。
- [ ] 更新 AI 路由规则：

  - 全市场、本周、本月、未来日期基金分红问题优先调用新扫描工具。
  - 已明确单只基金时继续优先调用 `fa_get_fund_dividend_profile`。
  - 裸历史列表继续使用 `fa_get_fund_dividend_history`。
  - 联接基金必须区分本基金直接分红和目标 ETF 资产层分红。

- [ ] 基金事件逐行“AI研判”设置 `assetType=fund`，携带基金类型、三类日期、每份金额、净值、比例状态、公告核验状态和 `as_of`。
- [ ] “AI扫描研判”调用新工具，只做全市场召回、排序和风险摘要，不在同一请求自动深挖多只基金。
- [ ] AI 提示词禁止把分配比例称为年化收益、禁止套用股票红利税档、禁止在公告未核验时声称“没有公告”。

## Phase 5 — 测试、验收与文档

- [ ] 新增根目录 `fund_dividend_feature_tests.php`，提供 fake Provider、fake NAV、内存缓存和分页 HTTP fixture。
- [ ] 覆盖解析：正常 JS 响应、空年度、畸形响应、字段不足、分页、超过最大页数和接口业务异常。
- [ ] 覆盖服务：日期过滤、跨年、提前停止、类型映射、份额类别去重、排序、分页、摘要、事件阶段。
- [ ] 覆盖安全比例：正常人民币、未知币种、净值缺失、净值日期等于/晚于除息日、历史事件、最低比例筛选和空值排序。
- [ ] 覆盖降级：事件 fresh/stale 缓存、类型映射失败、NAV 部分失败、熔断、负缓存和缓存防击穿。
- [ ] 覆盖详情：当前事件选择、公告精确匹配、公告未匹配、公告失败、联接基金说明、历史净值窗口及未来事件后置数据缺失。
- [ ] 覆盖 API：`asset_type` 默认兼容、参数白名单、非法基金代码、超过 60 日窗口和股票响应零变化。
- [ ] 覆盖前端：类型状态隔离、请求中止、移动端渲染、分页、详情焦点管理、HTML/属性转义。
- [ ] 覆盖 AI：工具严格 Schema、执行成功/失败、字符截断、中文状态、研究覆盖和路由提示词。
- [ ] 回归命令：

  - `.\php\php.exe ai_tool_runtime_tests.php`
  - `.\php\php.exe dividend_feature_tests.php`
  - `.\php\php.exe fund_dividend_feature_tests.php`
  - 需要真实数据时追加 `--live`
  - 双 PHP 服务就绪后追加 `--loopback`

- [ ] 验收标准：

  - 现有股票分红 47 项真实回归继续全部通过。
  - 基金默认 14 日窗口能展示已公告事件及三类日期。
  - 无净值、未知币种或上游部分失败时仍保留基础事件并清楚标记。
  - 冷缓存常规 14 日请求目标不超过 6 秒；事件和类型映射缓存命中后目标不超过 1 秒。
  - 列表净值按最多 50 只一批请求，不产生逐基金 N+1 调用。
  - 基金响应中不存在股票税档、`net_yield_pct` 或“每股”字段。
  - 手机和桌面均可完成筛选、翻页、详情和 AI 操作。
  - SSE 正常结束于 `[DONE]`，基金扫描工具只执行一轮且 `tool_call_finished.success=true`。

- [ ] 更新 README、`config.example.php`、API action 注释和数据授权说明；注明东方财富为未承诺 SLA 的公开页面源，商业化前需确认展示与再分发授权。
- [ ] 部署后验证股票/基金切换、基金日历 API、公告链接和 AI loopback；本功能不要求新增端口、数据库或 Nginx 规则。
