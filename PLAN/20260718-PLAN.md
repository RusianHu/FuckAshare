# 统一自选中心 + 数据状态条重构 Spec

## 1. 目标与固定决策

将现有“股票自选、基金自选、实时看板”重构为统一的个人工作区，并为全部投资数据请求建立可解释的数据状态体系。

固定决策：

- 本地优先，不建设账号、数据库或跨设备同步。
- 首版支持股票、基金；“实时盯盘”改为股票自选项的 `monitor` 属性。
- 采用“独立自选页＋全局快捷抽屉”。
- 每项属于一个自定义分组，同时支持多个标签、置顶和备注。
- 数据状态采用“导航下状态细条＋模块局部徽标＋详情层”。
- 覆盖行情、资金、策略、板块、分红、资讯、雪球、基金等投资数据；AI 对话、工具流、会话创建和静态资源不纳入。
- 允许拆分当前大型 `main.js`，新模块仍放项目根目录，不新增无必要的目录层级。
- 不引入新的外部行情源；先修复现有接口元数据和部分结果表达。

### [x] Phase 1 — 统一数据响应契约

- [ ] 扩展 `DataSourceResult`，成功与失败响应都明确输出 `status` 和规范化的 `meta.data_status`。
- [ ] 为旧兼容入口增加 `format=envelope` 模式；默认旧格式暂时不变，新前端统一请求 envelope。
- [ ] 修复股票行情、基金估值、热门股票、板块资金等入口丢弃 `source/meta/fallback` 的问题。
- [ ] 批量接口必须输出请求数、返回数和缺失代码，不再把“少返回几项”误判成完整成功。
- [ ] `null`、未返回和解析失败保持缺失状态；真实数值 `0` 仍按有效数据展示。
- [ ] 缓存读取增加 `cache_age_seconds`；不能取得上游数据时间时，`data_at` 必须为 `null`，不得用服务器当前时间冒充行情时间。
- [ ] 保留现有 `success/data/message/total` 等兼容字段，不改变现有 HTTP 状态策略。

统一响应核心结构：

```json
{
  "success": true,
  "source": "eastmoney",
  "action": "quote",
  "status": "success",
  "data": [],
  "meta": {
    "request_id": "短请求标识",
    "provider_status": 200,
    "cache": "miss",
    "cache_backend": "file",
    "cache_age_seconds": 0,
    "updated_at": "兼容字段",
    "observed_at": "本次响应完成时间",
    "data_at": null,
    "fallback_trace": [],
    "data_status": {
      "severity": "ok",
      "freshness": "fresh",
      "completeness": "complete",
      "route": "primary",
      "counts": {
        "expected": 2,
        "returned": 2,
        "missing": []
      },
      "warnings": []
    }
  }
}
```

枚举约束：

- `severity`：`ok | info | warning | error`
- `freshness`：`fresh | cached | stale | unknown`
- `completeness`：`complete | partial | empty | unknown`
- `route`：`primary | fallback | failed`
- `warnings[]`：`{ code, message }`
- 无可用数据时保留 `code/message`，`data_status.route=failed`。

### [x] Phase 2 — 统一自选存储与无损迁移

- [ ] 使用唯一主键 `stock:<规范代码>`、`fund:<6位代码>` 去重。
- [ ] 新存储键采用 `fa_watch_center_v2`，保存 schema 版本、revision、分组、项目和设置。
- [ ] 首次加载自动合并：
  - `fa_watchlist` → 股票自选。
  - `fa_realtime_codes` → 股票自选且 `monitor=true`。
  - `fa_fund_watchlist` → 基金自选。
  - 重复股票合并为一项，监控属性取逻辑或。
- [ ] 所有迁移项进入“默认分组”；名称缺失时先保留代码，后续刷新成功再补全。
- [ ] 迁移成功前不删除旧键；上线兼容期内由新存储单向镜像旧数组，保证旧版本可以回滚读取。
- [ ] 新数据损坏时先备份原字符串，再尝试从旧键恢复；仍失败则进入内存模式并向用户提示“本地保存不可用”。
- [ ] 监听浏览器 `storage` 事件，实现同浏览器多标签页同步；冲突采用 revision 较高者覆盖，revision 相同则以较新的 `updated_at` 为准。
- [ ] 完成 JSON 备份导出、JSON 合并导入、JSON 替换导入和 CSV 只读导出。
- [ ] 替换导入必须先完成全量校验并二次确认；任何错误都不得破坏现有数据。
- [ ] 导入后输出新增、更新、跳过、失败数量及逐项原因。

存储结构：

```js

  updatedAt: "ISO-8601",

    stockRefreshSeconds: 30,
    fundRefreshSeconds: 60

  groups: [{
    id: "default",
    name: "默认分组",
    sortOrder: 0,
    createdAt: "ISO-8601"
  }],
  items: [{
    id: "stock:sh600519",
    type: "stock",
    code: "sh600519",
    name: "贵州茅台",
    groupId: "default",
    tags: ["核心"],
    note: "",
    pinned: false,
    monitor: true,
    sortOrder: 0,
    createdAt: "ISO-8601",
    updatedAt: "ISO-8601"

```

限制：

- 最多 500 个项目、50 个分组。
- 每项最多 10 个标签，标签和分组名称最长 20 字符。
- 备注最长 500 字符。
- 基金不允许开启股票式实时监控，但仍可在页面可见时刷新估值。

### [x] Phase 5 — 数据状态中心与状态条

- [ ] 所有用户可见投资数据请求改由统一 `ApiClient` 发起。
- [ ] 请求必须声明 `scope`、中文名称、所属页面、是否影响全局状态、期望刷新周期和重试入口。
- [ ] 状态中心维护 `idle/loading/ok/notice/warning/error` 生命周期。
- [ ] 用户切换页面后，状态条默认只汇总当前页面；打开自选抽屉时额外合并抽屉状态。
- [ ] 被新请求替代、主动取消或页面切换导致的 Abort 不得计为失败。
- [ ] 导航下状态细条固定表达：
  - 正常：数据正常、数据项数量、最近完成时间。
  - 加载：正在更新及当前请求数。
  - 提醒：缓存、备用源或部分缺失数量。
  - 异常：失败数量，同时明确其他可用数据仍被保留。
  - 离线：浏览器离线，不清空最近成功数据。
- [ ] 聚合优先级固定为：离线 → 无可用数据错误 → 陈旧 → 部分/备用源 → 加载 → 缓存 → 正常。
- [ ] 模块标题处常驻紧凑状态按钮；正常时只显示状态点与时间，异常时显示“部分、降级、陈旧、失败”等文字。

- [ ] 详情按模块列出来源、请求完成时间、真实数据时间、缓存状态、完整度、缺失代码、警告和重试按钮。

- [ ] 提供“复制诊断信息”，只复制请求标识、模块、状态、时间、来源和脱敏错误码。
- [ ] 搜索联想等瞬时请求只在其控件内提示；AI SSE、AI 工具、`create_session.php` 和静态资源保持独立状态逻辑。

## 4. 现有接口与外部渠道评估

- 股票：现有 `MarketDataService` 已具备东方财富主源、雪球备用源和缓存降级能力，足以支持统一中心。
- 基金：现有搜索、估值、详情、排行和历史接口足够；首版接受估值数据源较单一的事实，通过状态条明确失败或陈旧，不伪造备用数据。
- 接口核心缺口不是新数据源，而是兼容 API 丢弃元数据、批量接口不声明缺失项，本计划优先修复这两点。
- 精确交易日历和节假日状态当前没有可靠统一接口，因此首版不显示确定性的“开盘/休市”结论；只有真实上游时间时才展示数据时间。
- 后续如需精确市场状态，可接入交易所官方日历或独立交易日历适配器；它不阻塞本次建设。
- 不采用浏览器抓取或页面解析作为状态条基础来源，避免 WAF、页面改版和时间语义不可靠。
- 服务端同步、匿名同步码或账号系统仅保留存储适配器扩展位，不在本次实现范围内。

## 5. 默认假设

- 未返回的第二轮产品选择按推荐值执行：独立页＋快捷抽屉、单组＋多标签、细状态条＋局部徽标＋详情层。
- 默认分组不可删除，可重命名；系统视图不可重命名或排序。
- 股票监控只代表自动刷新和监控视图归类，不包含交易提醒、推送或自动交易。
- “刷新时间”默认表示接口响应/缓存时间；只有 `data_at` 非空时才标注为“数据时间”。
- 导入默认使用合并模式；替换模式必须由用户明确选择并确认。
- 本地数据是唯一事实来源，行情快照和请求状态不进入长期持久化。
